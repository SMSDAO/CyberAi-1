## Brand Voice

### Tone
- **Technical but Approachable**: Speak to developers without being intimidating
- **Efficient**: Get to the point, value the user's time
- **Empowering**: Make users feel capable and in control
- **Professional**: Maintain credibility and trust

### Writing Style

**Do's**:
- ‚úÖ Use active voice
- ‚úÖ Be concise and clear
- ‚úÖ Use command examples
- ‚úÖ Explain benefits, not just features
- ‚úÖ Use developer-friendly language

**Don'ts**:
- ‚ùå Don't use marketing jargon
- ‚ùå Don't oversell or hype
- ‚ùå Don't use complex terminology without explanation
- ‚ùå Don't assume prior knowledge

**Example Copy**:

Good ‚úÖ:
> "Merge PRs from your phone with `/terminal merge`. No laptop required."

Bad ‚ùå:
> "Leverage our revolutionary mobile-first paradigm shift to synergistically optimize your pull request workflow."

## Messaging Framework

### Core Value Proposition
**For mobile developers** who need to manage GitHub repositories on the go, **GitHub Mobile Terminal** is a **command-based GitHub App** that lets you control PRs via terminal-style commands. Unlike the standard GitHub mobile app, Terminal provides **power-user efficiency** through a **familiar command-line interface**.

### Key Messages

1. **Mobile-First Control**
   - "Control your repository from anywhere"
   - "PR management without a laptop"
   - "Terminal commands in GitHub PRs"

2. **Developer Experience**
   - "Familiar terminal interface"
   - "Command-line power on mobile"
   - "Built by developers, for developers"

3. **Efficiency**
   - "Quick actions via comments"
   - "No context switching"
   - "Type commands, get results"

4. **AI-Powered** (v0.2+)
   - "SmartBrain AI reviews your code"
   - "Automated bug fixing"
   - "Intelligent suggestions"

### Target Audiences

**Primary**: 
- Mobile-first developers
- Open-source maintainers
- DevOps engineers
- Remote teams

**Secondary**:
- Engineering managers
- Solo developers
- Contract developers
- Indie hackers

## Partner & Co-Marketing Guidelines

### When Partnering

**Recommended Format**:
> "Built with CyberAi Terminal" or "Integrated with CyberAi Terminal"

**Logo Usage**:
- Must maintain aspect ratio
- Minimum size: 32px height
- Clear space: 10px on all sides
- Can use on light or dark backgrounds

**Co-Branding**:

### Attribution

When featuring Terminal in partner materials:
- Use official product name: "GitHub Mobile Terminal by CyberAi"
- Link to official GitHub App page
- Use approved logos and assets

## Brand Don'ts

**Don't**:
- ‚ùå Change the product name (use "GitHub Mobile Terminal" or "Terminal")
- ‚ùå Modify the logo or icons
- ‚ùå Use unapproved color variations
- ‚ùå Combine branding with competitor logos
- ‚ùå Make false or exaggerated claims
- ‚ùå Use Terminal branding for unrelated products
- ‚ùå Suggest official endorsement without permission
